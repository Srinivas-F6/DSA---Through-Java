Parent Node: A node that is an immediate predecessor of another node. Example: 35 is the parent of 3 and 6.
Child Node: A node that is an immediate successor of another node. Example: 3 and 6 are children of 35.
Root Node: The topmost node in a tree, which does not have a parent. Example: 15 is the root node.
Leaf Node (External Node): Nodes that do not have any children. Example: 1, 10, 12, 5, 7, 7 are leaf nodes.
Ancestor: Any node on the path from the root to a given node (excluding the node itself). Example: 15 and 35 are ancestors of 10
Descendant: A node x is a descendant of another node y if y is an ancestor of x. Example: 1, 10, and 6 are descendants of 35.
Sibling: Nodes that share the same parent. Example: 1 and 10 are siblings, and 5 and 7 are siblings.
Level of a Node: The number of edges in the path from the root to that node.The root node is at level 0.
Internal Node: A node with at least one child.
Neighbor of a Node: The parent or children of a node.
Subtree:  A node and all its descendants form a subtree.

Tree :
 A Tree is a non-linear hierarchical data structure made of nodes connected by edges, where one node is called the root and each node can have zero or more child nodes.

Properities of the Tree :
  Root Node â†’ The topmost starting node of a tree.
  Parent Node â†’ A node that has child nodes.
  Child Node â†’ A node connected below a parent node.
  Leaf Node â†’ A node with no children.
  Internal Node â†’ A node having at least one child.
  Edge â†’ Connection between two nodes.
  Degree of Node â†’ Number of children a node has.
  Degree of Tree â†’ Maximum degree among all nodes in the tree.
  Level of Node â†’ Position of a node from the root.
  Depth of Node â†’ Number of edges from root to that node.
  Height of Tree â†’ Length of longest path from root to deepest node. (Height always measures from the leaf node)
  Subtree â†’ A smaller tree formed from a node and its descendants.
  Path â†’ Sequence of connected nodes.
  Ancestor â†’ Nodes above a particular node.
  Descendant â†’ Nodes below a particular node.
  Edges Formula â†’ A tree with n nodes has n - 1 edges.

Binary Tree :
 A Binary Tree is a special type of tree in which each node can have at most two children, called the left child and right child.

Properities of Binary Tree :
  Maximum Children â†’ Each node can have at most 2 children.
  Left Child â†’ Child node on the left side.
  Right Child â†’ Child node on the right side.
  Maximum Nodes at Level l â†’ 2^l nodes can exist at level l.
  Maximum Nodes in Height h â†’ 2^(h+1) - 1 nodes.
  Minimum Height â†’ A balanced binary tree has minimum possible height.
  Binary Tree Types â†’ Full, Complete, Perfect, Balanced, Degenerate.
  Traversal Methods â†’ Preorder, Inorder, Postorder, Level Order.
  Recursive Structure â†’ Every subtree is also a binary tree.
  Edges Formula â†’ For n nodes, edges = n - 1.

Types of Binary Trees :
  Full Binary Tree â†’ A binary tree in which every node has either 0 or 2 children.
  Complete Binary Tree â†’ A binary tree in which all levels are completely filled except possibly the last level, and the last 
                        level nodes are filled from left to right.
  Perfect Binary Tree â†’ A binary tree in which all internal nodes have exactly 2 children and all leaf nodes are at the same level.
  Degenerate (Skewed) Binary Tree â†’ A binary tree in which every parent node has only one child, making it similar to a linked list.

Binary Search Tree (BST) :
 A Binary Search Tree (BST) is a special type of binary tree in which:
  all values in the left subtree are smaller than the root node, and
  all values in the right subtree are greater than the root node.

Properities of Binary Search Tree :
  BST Rule â†’ Left subtree values are smaller and right subtree values are greater than the root.
  Binary Tree Property â†’ Each node has at most 2 children.
  Inorder Traversal â†’ Gives elements in sorted order.
  Search Operation â†’ Faster compared to normal binary tree.
  Average Time Complexity â†’ Search, Insert, Delete â†’ O(log n) in balanced BST.
  Worst Case â†’ BST becomes skewed and operations become O(n).
  No Duplicate Values â†’ Usually duplicates are not allowed in BST.
  Recursive Nature â†’ Left and right subtrees are also BSTs.
  Minimum Value â†’ Leftmost node contains minimum value.
  Maximum Value â†’ Rightmost node contains maximum value.
